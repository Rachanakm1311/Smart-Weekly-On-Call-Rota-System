1
Introduction & Login
Set the scene — what problem does this solve?
0:00 – 0:25
🎤 Say
"This is the Smart Weekly On-Call Rota Management System — built on ASP.NET Core MVC and SQL Server for IBM's application support teams. The problem it solves: no more spreadsheets, no more manual chasing. It automates weekly on-call scheduling, respects regional public holidays, handles leave and shift swaps, sends Friday email reminders, and gives every engineer instant access to IBM's ICA resolution tool during P1 and P2 incidents."
🖥 Screen Actions
  • Start on the Login page — show both Admin and Employee login options
  • Enter Admin credentials and click Login
  • Let the Admin Dashboard fully load before moving on
2
Admin Dashboard — Central Command
Show the overview and employee management
0:25 – 0:50
🎤 Say
"The Admin Dashboard shows a live snapshot — active teams, total employees, pending leave requests, and pending shift swaps. The current week's on-call schedule is right here with the Primary and Backup engineer for each team — including their phone numbers, which we'll use later for ServiceNow integration. Employees are assigned an IBM India region — Bangalore, Hyderabad, Pune, Chennai, Kolkata, or Delhi — which drives the holiday logic."
🖥 Screen Actions
  • Point to KPI cards — Teams, Employees, Pending Leaves, Pending Swaps, Current On-Call
  • Scroll to the Current Week On-Call table — hover over the Primary column to show phone number beneath the name
  • Click Employees in the sidebar — show the list with the new Region badge (e.g. Bangalore, Kolkata)
  • Click Edit on one employee — show the Region dropdown (12 IBM India offices)
  • Press Cancel and return to Dashboard
Tip: Pause briefly on the phone number in the on-call table — it's a key detail for the ServiceNow story.
3
Rota Generation — Automatic & Manual
The core engine — show both modes
0:50 – 1:30
🎤 Say
"There are two ways the rota gets generated. Automatically — every Friday at 8 AM, a background service runs and generates next week's schedule for all teams, then immediately emails the assigned engineers. Manually — an Admin can generate a rota for any team and any week right here by clicking 'Generate Weekly Rota'. The system uses a Rotation Queue — it picks the next person in line as Primary and the person after as Backup, automatically skipping anyone on approved leave or on a regional holiday. After each week, the Primary engineer moves to the back of the queue — completely fair rotation."
🖥 Screen Actions
  • Click OnCall Schedule in the sidebar — show the schedule table with phone numbers visible
  • Click Generate Weekly Rota button — the modal opens
  • Select a team from the dropdown, pick a week date — point out: "This is the manual override"
  • Click Cancel (don't actually generate to keep data clean)
  • Click Rotation Queue in the sidebar — show the position order for one team
  • Say: "Queue position 1 becomes Primary, position 2 becomes Backup — next week they rotate"
Tip: Mention: "The automatic Friday run means the admin never has to remember to do this — it just happens every week."
4
Holiday Calendar & Region-Aware Skipping
A key differentiator — regional public holidays
1:30 – 1:55
🎤 Say
"Here's something unique. IBM India has regional holidays — Ugadi in Bangalore, Durga Puja in Kolkata, Bonalu in Hyderabad, and so on. When the rota generator runs, it checks whether the week contains a holiday in an employee's region. If it does, that employee is automatically skipped and the next available person is picked instead. But — if an engineer volunteers to work on that holiday, they can submit a 'Holiday Work' leave request, confirm their comp-off date, and once approved, they stay in the rotation pool."
🖥 Screen Actions
  • Click Holiday Calendar in the sidebar — show the list of regional holidays with Location column (Bangalore, Kolkata, Hyderabad…)
  • Switch to Employee portal (second browser tab)
  • Click Apply Leave → change Leave Type to "Holiday Work (Comp-Off)"
  • Show the amber panel that appears: confirmation checkbox + comp-off date picker
  • Say: "Once approved, the rota generator treats this employee as available despite the holiday"
  • Press Cancel and return
Tip: This segment differentiates the system from a basic scheduler — linger here for a second.
5
Employee Portal & Friday Email Notification
Self-service + automated communication
1:55 – 2:30
🎤 Say
"Every employee has their own portal. Their dashboard tells them instantly whether they're Primary, Backup, or not scheduled this week. They can see the full team schedule with contact numbers for the on-call engineers — useful when they need to escalate. They can apply for leave, request a shift swap with a colleague, and manage notifications. And every Friday at 8 AM, the system automatically sends an email to next week's Primary and Backup engineers with their on-call dates — so they're never caught off guard."
🖥 Screen Actions
  • On the Employee Dashboard — point to the on-call role banner ("You are PRIMARY on-call this week")
  • Scroll to the This Week's On-Call card — show Primary Phone and Backup Phone links
  • Click Who Is On-Call — show the full schedule table; point out phone numbers under each name
  • Click Notifications — show a "WeeklyReminder" notification: "You are the PRIMARY on-call engineer for week…"
  • Say: "This same message is also sent as an email every Friday — both in-app and via Gmail"
  • Click Leave Requests → Apply — briefly show the Holiday Work option
Tip: Pre-load the Notifications page in advance so there's at least one WeeklyReminder visible.
6
ICA Tool — P1 / P2 Incident Resolution
The AI-powered resolution assistant
2:30 – 2:50
🎤 Say
"When a P1 or P2 incident fires, the on-call engineer needs to act fast. We've integrated IBM's ICA tool — the Intelligent Curation Assistant — directly into the portal. Every page has a blue robot button in the bottom corner. Click it, and a drawer slides in with the ICA chat for your specific team. The engineer pastes the error message or application logs, and ICA suggests resolution steps based on past incidents and runbooks. We've already set up the MS-COTS team agent — and as the other teams create their ICA agents, we just drop in the new URL. No code changes needed."
🖥 Screen Actions
  • Click the blue robot button (bottom-right of screen) — the ICA drawer slides in smoothly
  • Point to the drawer header: "Agent for MS-COTS · P1 / P2 resolution"
  • Read the 5-step guide briefly — paste error, get steps, document findings, escalate to Backup
  • Click Open ICA Tool — show it opening in a new tab
  • Close the tab, close the drawer (Escape key)
  • Switch to Admin portal — open the ICA drawer there, show the team picker dropdown (Admin can select any team)
Tip: Mention: "In the Admin portal, the drawer has a team selector — so the admin can open the right agent for whichever team is having the incident."
7
Closing Summary
Wrap up with value and tech stack
2:50 – 3:00
🎤 Say
"To summarise — automated and manual rota generation with fair rotation, region-aware holiday skipping, comp-off support, Friday email reminders, phone numbers for ServiceNow integration, and the ICA tool one click away for every on-call engineer. Built on ASP.NET Core 10, Entity Framework Core, SQL Server, and Bootstrap 5. Thank you."
🖥 Screen Actions
  • Return to the Admin Dashboard — a clean, full-screen ending shot
